# MoneyMovers - Transferência de Dinheiro 

O **MoneyMovers** é uma aplicação financeira focada na simplicidade e segurança, permitindo aos utilizadores enviar e receber dinheiro de forma rápida, segura e sem taxas.

Este repositório contém todo o planeamento ágil, estruturação de dados e fluxo de trabalho realizado durante o desenvolvimento do projeto através do Trello.

---

## Propósito do Sistema
> "MoneyMovers é uma aplicação simples que permite aos utilizadores enviar e receber dinheiro de forma rápida, segura e sem taxas."

## Arquitetura e Design (UML & UI/UX)

O projeto seguiu uma base técnica sólida antes da implementação:

* **UML:** Definição de **Conceptual ERD** (Modelo Entidade-Relacionamento) e **Class Diagram** para garantir a integridade dos dados.
* **UI/UX:** Mapeamento completo de wireframes para a Feature A (Enviar) e Feature B (Receber), cobrindo desde a página inicial até ao ecrã de sucesso.
* **Tech Stories:** Foco em segurança com encriptação de dados de pagamento e integração de APIs bancárias.

## Funcionalidades (User Stories)

Dividimos o projeto em três grandes eixos, utilizando **Story Points** (Fibonacci) para estimar o esforço:

### Feature A - Enviar Dinheiro
* Adicionar IBAN de amigos.
* Seleção de montantes e confirmação de segurança.
* Gestão de contactos favoritos.

### Feature B - Receber Dinheiro
* Notificações em tempo real.
* Identificação de remetentes e gestão de recebimentos (Aceitar/Recusar).
* Configuração de segurança do IBAN próprio.

### Feature C - Histórico e Notificações
* Lista de transações detalhada.
* Filtros inteligentes por tipo de transferência e pesquisa por nome.

## Metodologia Ágil (Scrum & Kanban)

Utilizámos um quadro **Kanban** dinâmico para monitorizar o progresso da **Sprint 1 (MVP)**.

### Ciclo de Vida das Tarefas:
1.  **Backlog:** Ideias e requisitos iniciais.
2.  **To Do (Sprint 1):** Prioridades do MVP (ex: Adicionar IBAN, Confirmar Transferência).
3.  **Doing:** Tarefas em desenvolvimento.
4.  **Blocked:** Impedimentos técnicos.
5.  **Tests / Code Review:** Garantia de qualidade.
6.  **Done:** Funcionalidades finalizadas e validadas.
